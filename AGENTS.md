# Sun CLI 工作流程规范

## 流程概述

为确保开发过程的清晰性和可控性，Sun CLI 采用以下工作流程：

1. **讨论阶段**：用户提出需求或问题
2. **规划阶段**：双方讨论实现方案
3. **执行阶段**：用户明确授权后执行操作
4. **验证阶段**：检查执行结果并确认完成

## 详细流程

### 1. 讨论阶段
- 用户提出具体需求或问题
- 助手分析需求并提供初步建议
- 双方讨论实现细节和可能的方案
- 明确任务目标和预期结果

### 2. 规划阶段
- 助手根据讨论结果制定详细的实现计划
- 明确需要修改的文件和具体改动
- 评估可能的风险和影响
- 确认所需的工具和资源

### 3. 执行阶段
- 助手等待用户的明确执行指令
- 用户确认方案后，明确授权执行
- 助手按照计划执行操作
- 执行过程中保持透明，及时反馈进度

### 4. 验证阶段
- 执行完成后，助手提供详细的执行报告
- 检查执行结果是否符合预期
- 确认所有修改都已正确实施
- 总结本次任务的完成情况

## 执行准则

1. **明确授权**：所有修改操作必须在用户明确授权后执行
2. **透明操作**：执行过程中要保持透明，及时报告进度
3. **安全第一**：确保所有操作的安全性，避免潜在风险
4. **质量保证**：执行完成后要验证结果，确保质量
5. **文档记录**：重要操作要记录在文档中，便于后续参考

## 适用场景

- 代码修改和优化
- 配置文件调整
- 工具集成和扩展
- 系统维护和升级

## 注意事项

- 对于高风险操作，需要特别谨慎
- 执行前要备份重要数据
- 执行后要进行充分测试
- 保持沟通渠道畅通，及时响应用户反馈
