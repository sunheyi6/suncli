## 模块职责

| 模块 | 职责 | 新增/改造 |
|------|------|----------|
| `ChatSession` | 核心对话循环、工具调度、上下文压缩 | 集成 Nudge + Skill v2 |
| `ToolExecutor` | 19 种工具解析与执行 | 新增 `skill_view` / `skill_manage` |
| `MemoryManager` | 跨会话记忆持久化 | 增加容量限制 + 安全扫描 |
| `SkillManagerV2` | 经验手册管理 | **新增** |
| `NudgeEngine` | 计数器 + 后台审查触发 | **新增** |
| `ReviewAgent` | 静默分析对话快照 | **新增** |
| `SecurityScanner` | Memory/Skill 威胁检测 | **新增** |
| `PromptManager` | 系统提示词构建 | Skill 索引渐进加载 |

## 数据流向

### 正常对话流

1. 用户输入 → `stream_message()`
2. `NudgeEngine.on_user_turn()` 计数器 +1
3. `_run_tool_loop()` 最多 10 轮工具调用
4. 每轮工具调用后 `NudgeEngine.on_tool_iteration()` 计数器 +1
5. 无工具调用 → 返回最终响应

### 后台学习流

1. 最终响应发送给用户后
2. `maybe_trigger_review()` 检查计数器是否达阈值
3. 达阈值 → `asyncio.create_task()` 启动后台 Review Agent
4. Review Agent 分析对话 → 决定是否 save_memory / skill_manage
5. 用户完全无感知

### Skill 使用流

1. 系统提示词注入 **Skill 索引**（仅名字+描述）
2. Agent 判断当前任务与某 Skill 相关
3. Agent 调用 `skill_view(name)` 加载全文
4. Agent 按 Skill 的 Steps 执行，参考 Pitfalls 避坑
5. 完成后 `SkillManagerV2.record_usage()` 更新生命周期
